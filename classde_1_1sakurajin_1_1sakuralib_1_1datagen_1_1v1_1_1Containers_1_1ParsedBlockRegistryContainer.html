Detailed Description

A ParsedBlockRegistryContainer is a Block Registry Container that works with the DatagenAPI.

This connects the owo registration system with the DatagenAPI and allows generating Blockstates, Models, Recipes and more during Runtime. If a Block does not Implement the Datagen Interfaces, it will only be registered and the DatagenAPI will ignore it. All you have to do is create all your Block as static public variables and implement a constructor with no parameters that passes a DatagenModContainer to the super constructor.

The following parser are added by default:

  • BlockModelGenerationParser
  • BlockStateGenerationParser
  • BlockItemGenerationParser
  • ItemModelGenerationParser
  • RecepieGenerationParser
  • LootTableParser
  • TagGenerationParser

You can add your own parsers to the DatagenModContainer to add your own annotations or handle interface implementations.

See also
ParsedContainerBase for more information about adding Parsers
